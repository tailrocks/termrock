TermRock components accept consumer-owned wording and domain state while owning reusable terminal rendering, focus behavior, hit geometry, Unicode safety, narrow-terminal behavior, and non-color cues.

| Component | Purpose |
|---|---|
| [ActionBar](./action-bar) | A horizontal group of stable, caller-owned actions with painted hit regions. |
| [Backdrop](./backdrop) | A configurable themed fill painted behind modal content. |
| [ChoiceDialog](./choice-dialog) | A modal choice prompt with stable action identities and canonical traversal. |
| [DetailTable](./detail-table) | A selectable key/value table with stable rows and typed activation capabilities. |
| [Dialog](./dialog) | A framed modal surface with semantic chrome and caller-owned content. |
| [DiffView](./diff-view) | A vertically scrollable, syntax-neutral presentation of projected diff lines. |
| [Form](./form) | A responsive form layout with stable focus, validation, and hit geometry. |
| [HintBar](./hint-bar) | A wrapping row of prioritized keyboard hints with semantic styling. |
| [List](./list) | A selectable, scrollable list over borrowed rows and stable identities. |
| [LogPane](./log-pane) | A bounded, scrollable log buffer with freeze-on-scroll and tail following. |
| [MessageDialog](./message-dialog) | A message dialog composed with optional scrollable detail rows. |
| [Panel](./panel) | A themed bordered container with semantic focus emphasis. |
| [Picker](./picker) | A filterable stable-ID list composition with caller-owned matching and ordering. |
| [Progress](./progress) | A deterministic determinate bar or caller-ticked indeterminate indicator. |
| [SplitPane](./split-pane) | A resizable two-pane layout with bounded ratios and collapse support. |
| [StatusBar](./status-bar) | A one-row collection of prioritized, interactive status slots. |

| [Viewport](./viewport) | A two-axis scrollable view over borrowed terminal lines. |

## Shared guarantees

All widgets tolerate empty and narrow rectangles. Disabled items are not hit targets. Mouse regions derive from painted geometry. Stable identities survive filtering and reordering. Consumers own effects, navigation policy, and dialog stacking.

Panel and dialog borders use one plain, single-line glyph set. Focus is communicated through the semantic `Role::BorderFocused` theme role; inactive containers use `Role::Border`.
