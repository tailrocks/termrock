---
title: Interaction conventions
---

Tabs use one focus stop with arrow navigation. Their painted label and active-indicator rows share the same hit geometry. Disabled items are not hit targets. Hover never moves keyboard focus. Scroll only over the owning region. Dialog geometry and inside/outside classification are pure; `DialogSpec` expresses independent horizontal and vertical margins, and applications own modal stacks. `bottom_rows` provides one tiny-terminal collapse contract for product-defined bottom chrome slots. Hints advertise primary bindings and hide aliases. Navigation uses stable IDs, never durable indices.
