# 0008 — `v0.9.0` semantic theme palette

## Boundary

`v0.9.0` removes donor-product color tokens and raw preset internals from the
crate root. Consumer styling now flows through `Theme` and `Role`. No root
aliases or compatibility constants are provided.

## Old to new

| Removed surface | Canonical replacement | Required consumer edit |
|---|---|---|
| `termrock::Rgb` | `termrock::style::Rgb` | Update the import when constructing a Ratatui color with `style::color`. |
| `termrock::PHOSPHOR_GREEN` | `termrock::style::PHOSPHOR_GREEN` | Use only when the documented preset color itself is required; prefer a semantic role. |
| `termrock::PHOSPHOR_DARK` | `termrock::style::PHOSPHOR_DARK` | Update the path, or prefer a semantic role. |
| `termrock::PREVIEW_CARD` | `termrock::style::PREVIEW_CARD` | Update the path for lookbook-style preview canvases. |
| Other phosphor palette constants | `Theme::tailrocks_phosphor()` and `Theme::style(Role)` | Replace raw palette access with the matching semantic role. |
| `RAIN_*`, `MENU_*`, `DEBUG_AMBER`, `STATUS_BLOCKED_RED`, `BRAND_BLOCK`, `LINK_BLUE` | Consumer-owned values | Define these product tokens in the consuming application if still required. |

Before:

```rust
use termrock::{PHOSPHOR_GREEN, RAIN_BODY};
```

After:

```rust
use termrock::style::PHOSPHOR_GREEN;

// RAIN_* were donor-product tokens. Define product colors in the application.
```

For component styling, prefer semantic roles:

```rust
use termrock::{Theme, style::Role};

let theme = Theme::default();
let focus_style = theme.style(Role::Focus);
```

## Ownership changes

TermRock owns semantic theme roles and its default preset. Consumers own
product-specific palettes and may override any role with `Theme::with_role` or
construct a complete palette with `Theme::from_fn`.

## Removed concepts

- donor-product animation, menu, debug, and status colors in TermRock;
- a raw RGB palette at the crate root;
- public widget implementation constants as a styling API.
