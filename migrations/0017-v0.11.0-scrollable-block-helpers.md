# 0017 — `v0.11.0` first-class scrollable block helpers

## Boundary

`v0.11.0` restores product-neutral scrollable-block composition helpers after
consumers still needed them. Helpers live under `termrock::scroll` and paint
through `Panel` + semantic roles rather than private layout facades.

## Old to new

| Previous surface | Canonical replacement | Required consumer edit |
|---|---|---|
| `layout::dialog_scroll_axes` | `scroll::dialog_scroll_axes` | Update the import path. |
| `layout::scroll_hint_spans` | `scroll::scroll_hint_spans` or `keymap::SCROLL_HINT_KEYMAP.hint_spans_for_axes` | Prefer the `scroll` free function for parity, or call the keymap directly. |
| `layout::DialogBodyScroll` | `scroll::DialogScroll` | Rename the type; construct with `DialogScroll::default()` / `::new()` then set public offsets. |
| `scroll::render_scrollable_block*` (removed in 0014) | Restored `scroll::render_scrollable_block` / `render_scrollable_block_at` | Re-import from `termrock::scroll`. Focus drives `PanelEmphasis::Focused`. |
| `scroll::render_lines_with_offset_in_area` (removed in 0014) | Restored under `termrock::scroll` | Re-import from `termrock::scroll`. |

Before:

```rust
use termrock::layout::{DialogBodyScroll, dialog_scroll_axes, scroll_hint_spans};
```

After:

```rust
use termrock::scroll::{DialogScroll, dialog_scroll_axes, scroll_hint_spans};
```

## Ownership changes

TermRock owns bordered scrollable-block painting and axis derivation for
one-cell panel geometry. Consumers own content lines, offsets, and which
surface is focused.
