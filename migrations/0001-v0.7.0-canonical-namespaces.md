# 0001 — `v0.7.0` canonical namespaces

## Boundary

`v0.7.0` intentionally removes the donor-shaped `components` facade. Consumers
must use the canonical namespaces before updating their exact revision pin.

## Old to new

| Before | New canonical surface |
|---|---|
| button strip | `termrock::widgets::ActionBar` |
| select list | `termrock::widgets::List` |
| tab strip | `termrock::widgets::Tabs` |
| text field/filter | `termrock::widgets::TextInput` composition |
| panel and hint row | `termrock::widgets::{Panel, HintBar}` |
| status footer | `termrock::widgets::{StatusBar, StatusBarState}` with caller-owned slot meaning |
| detail table | `termrock::widgets::DetailTable` with caller-owned stable row IDs and payloads |
| confirm/save-discard | `termrock::widgets::{ChoiceDialog, ChoiceDialogState}` with caller actions and outcomes |
| error/status popup | `termrock::widgets::{MessageDialog, DetailTableState, Toast}` |
| diff view | `termrock::widgets::DiffView` |
| modal geometry and scrolling | `termrock::{layout, interaction, scroll}` |
| terminal escapes | `termrock::osc` typed requests and pure encoders |

## Required edits

1. Remove every `termrock::components` import.
2. Import dialog geometry from `termrock::layout`.
3. Import focus, hover, modal lifecycle, and hit regions from
   `termrock::interaction`.
4. Import scrolling geometry/rendering from `termrock::scroll`.
5. Use `termrock::widgets::Backdrop` for backdrop policy;
   `Backdrop::default()` is the opaque terminal-background fill.
6. Wrap foreign receivers with `runtime::ClosureSubscription`; TermRock does not
   depend on Tokio.
7. Keep brand headers, row projection, lifecycle stacks, output policy, wording,
   effects, and application runtime helpers in the consumer.
8. Delete donor-shaped render helpers and compatibility imports after migration.

## Before and after

Before:

```rust
use termrock::components::{SelectList, StatusFooter};
```

After:

```rust
use termrock::widgets::{List, ListState, StatusBar, StatusBarState};
```
