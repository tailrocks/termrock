# 0019 — `v0.11.0` bounded `LogPane` scrollback

## Boundary

`LogPaneState::new()` now retains at most 10,000 lines. Unbounded retention is
an explicit opt-in. This removes the previous memory and render-cost footgun
while preserving append-oriented ownership, frozen scrollback, and tail follow.

## Old to new

| Previous behavior or surface | Canonical replacement | Required consumer edit |
|---|---|---|
| `LogPaneState::new()` retained every appended line | `LogPaneState::new()` retains `DEFAULT_LOG_HISTORY_LINES` | No edit when bounded history is acceptable; account for oldest-line eviction. |
| Implicit unbounded history | `LogPaneState::new().unbounded()` | Add `.unbounded()` only when the consumer owns an external retention policy and accepts unbounded memory and measurement cost. |
| Keyboard-only scrollback | `LogPaneState::scroll_by(delta)` | Route neutral wheel input to negative deltas for older lines and positive deltas for the live tail. |
| End-key handling was the only follow control | `LogPaneState::follow()` and `is_following()` | Resume follow explicitly from application controls when needed. |
| No oldest-line shortcut | `KeyCode::Home` through `handle_key` | Forward the neutral key event; Home now freezes at the oldest visible window. |
| Callers manually wrapped `styled_spans` in a line | `ansi_text::line_from_ansi(input, default_style)` | Parse ANSI once at ingestion, then append the returned owned line. |

## Before

```rust
use termrock::widgets::LogPaneState;

let mut logs = LogPaneState::new();
for line in incoming_lines {
    logs.append(line);
}
// History grew without a library bound.
```

## After: bounded default

```rust
use termrock::{
    ansi_text::line_from_ansi,
    style::{Role, Theme},
    widgets::LogPaneState,
};

let theme = Theme::default();
let mut logs = LogPaneState::new();
logs.append(line_from_ansi(
    "\u{1b}[31merror\u{1b}[0m: build failed",
    theme.style(Role::Text),
));

// Neutral ScrollUp input uses a negative, top-relative delta.
logs.scroll_by(-1);
logs.follow();
```

## After: explicit unbounded retention

```rust
use termrock::widgets::LogPaneState;

let mut logs = LogPaneState::new().unbounded();
for line in incoming_lines {
    logs.append(line);
}
```

## Ownership

`LogPaneState` remains the deliberate owned-data exception among projection
widgets. Logs arrive as a stream over time; the state atomically owns retention,
eviction, frozen-view distance, and follow transitions. Consumers continue to
own process execution, wording, severity policy, and terminal-event adaptation.

## Removed concepts

- unbounded retention as an implicit default;
- keyboard-only access to scrollback state;
- caller-side ANSI span-to-line boilerplate.
