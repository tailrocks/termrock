# 0020 — `v0.11.0` explicit `LogPane` oldest navigation

## Boundary

`LogPaneState::scroll_to_oldest()` exposes the same semantic transition as
Home without requiring consumers to synthesize a key event. The request is
valid before first render: state resolves the exact oldest offset after the
widget records viewport geometry.

## Old to new

| Previous surface | Canonical replacement | Required consumer edit |
|---|---|---|
| `handle_key(KeyCode::Home)` was the only oldest-window transition | `LogPaneState::scroll_to_oldest()` | Call the state method from buttons, commands, or restored application state; keyboard adapters may continue forwarding Home. |
| `scroll_by` before first render had no viewport geometry | `scroll_to_oldest()` records pending oldest intent | Use the semantic method when initial state must open at the oldest retained window. |

## Before

```rust
use termrock::input::{KeyCode, KeyEvent, KeyModifiers};

state.handle_key(KeyEvent::new(KeyCode::Home, KeyModifiers::NONE));
```

## After

```rust
state.scroll_to_oldest();
```

## Ownership

`LogPaneState` owns scrollback position and delayed resolution against painted
viewport geometry. Consumers own commands, controls, and neutral input routing.
