# 0016 — `v0.11.0` ordinary vs strong text and Viewport emphasis

## Boundary

`v0.11.0` separates ordinary body text from strong/heading text in the semantic
theme, and makes focused borders a first-class `Viewport` control. Consumers no
longer invent bold white styles or override `Role::Border` merely to paint the
active scroll container.

## Old to new

| Previous surface | Canonical replacement | Required consumer edit |
|---|---|---|
| `Role::Text` as bold heading/body text | `Role::Text` (ordinary) + `Role::TextStrong` (bold) | Use `Role::Text` for default-weight labels and body copy. Use `Role::TextStrong` for titles, keys that need weight, and headings. |
| `theme.style(Role::Text).add_modifier(Modifier::BOLD)` | `theme.style(Role::TextStrong)` | Drop the manual bold modifier when the intent is strong text. |
| `Theme::default().with_role(Role::Border, theme.style(Role::BorderFocused))` for an active `Viewport` | `Viewport::new(lines, theme).emphasis(PanelEmphasis::Focused)` | Pass emphasis on the widget; keep the shared theme untouched. |
| Panel/dialog titles relying on bold `Role::Text` | unchanged call sites; titles now resolve through `Role::TextStrong` | No consumer change unless you overrode `Role::Text` specifically to restyle titles. Override `Role::TextStrong` for title weight instead. |

Before:

```rust
use termrock::{Theme, style::Role, widgets::Viewport};

let theme = Theme::default().with_role(
    Role::Border,
    Theme::default().style(Role::BorderFocused),
);
let title = theme.style(Role::Text).add_modifier(Modifier::BOLD);
let viewport = Viewport::new(&lines, &theme).title("Build log");
```

After:

```rust
use termrock::{
    Theme,
    style::Role,
    widgets::{PanelEmphasis, Viewport},
};

let theme = Theme::default();
let title = theme.style(Role::TextStrong);
let viewport = Viewport::new(&lines, &theme)
    .title("Build log")
    .emphasis(PanelEmphasis::Focused);
```

## Ownership changes

TermRock owns the ordinary/strong text distinction and focused viewport border
selection. Consumers supply content, product wording, and which container is
the interaction owner.

## Removed concepts

- treating `Role::Text` as the only bold white style consumers could reuse;
- consumer-side theme surgery that remaps `Role::Border` solely to focus a
  scrollable viewport.
