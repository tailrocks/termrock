# 0014: Scroll and hover unification

TermRock now has one scroll-state family, one hover primitive, and no orphaned
free-function widget facades. This is a breaking, forward-only support-surface
cleanup.

## Removed dialog support surface

`layout::DialogBodyScroll`, `DialogBorder`, `render_scrollable_dialog_body`,
`render_dialog_shell`, `dialog_inner_chunks`, `dialog_inner_height`,
`dialog_scroll_axes`, and `scroll_hint_spans` were unused duplicate machinery.

Use `scroll::DialogScroll` for two-axis offsets, `widgets::Dialog`/`Panel` for
chrome, Ratatui `Layout` for caller composition, and
`keymap::SCROLL_HINT_KEYMAP.hint_spans_for_axes` for hints. `ScrollAxes`,
`ScrollAxis`, and `mouse_scroll_delta` now have only their canonical
`termrock::scroll` paths; the `layout` re-exports are gone.

```rust
// before
use termrock::layout::{DialogBodyScroll, ScrollAxes};
let scroll = DialogBodyScroll::new();

// after
use termrock::scroll::{DialogScroll, ScrollAxes};
let scroll = DialogScroll::default();
```

The removed raw-byte/SGR adapters had no runtime consumers. Convert terminal
events to `termrock::input::Event` and call the canonical scroll helpers.

## Removed render facades

The following zero-consumer rendering family is gone:

- `ScrollableList` (use `widgets::List` + `ListState`)
- `render_selected_lines_in_area` (use `widgets::List`)
- `render_lines_with_offset_in_area` (slice visible data, then render a
  Ratatui `Paragraph`)
- `render_scrollable_block` and `render_scrollable_block_at` (compose
  `widgets::Panel`/`Viewport`)

Scrollbar painting primitives remain available. `ScrollbarStyle` remains
because the canonical `List` uses it.

## Canonical scroll names

| Removed | Replacement |
|---|---|
| `rendered_max_line_width` | `max_line_width` (raw rendered width) |
| `rendered_max_offset` | `max_offset_u16` for `u16`, otherwise `max_offset` |
| `rendered_cursor_follow_offset` | `cursor_follow_offset` |
| `effective_offset_u16` | `effective_offset` |

The padded-width implementation belonged only to the deleted block facade; it
has no replacement. Callers must measure the content they actually paint.

## Hover

`HoverTracker` and its private `ClickableRect` record are removed. Use the
caching, `HitRegion`-native pair:

```rust
use termrock::interaction::{HitRegion, HoverState};

let regions = [HitRegion { id: "save", area }];
let mut hover = HoverState::default();
hover.update(pointer, &regions);
assert_eq!(hover.hovered(), Some(&"save"));
```

Widgets with state-owned input may continue exposing their own `hover`
methods; do not introduce another rectangle registry.

## Modal ownership

`ModalStack::dismiss_current` is removed because it could leave saved parents
without an active modal. Use `pop()` to restore one parent, `clear_chain()` to
dismiss the entire chain, or `take_current()` followed by explicit
`parents_mut()` management. Calling `open_sub` with no current modal now starts
a clean root and clears any stale parent chain.
