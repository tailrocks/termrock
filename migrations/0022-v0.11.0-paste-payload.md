# Migration 0022: paste payloads in the neutral event contract

## Removed surface

`termrock::input::Event::Paste` was a unit variant. It discarded the backend's
bracketed-paste text. `Event` also implemented `Copy`.

## Canonical replacement

`Event::Paste(String)` owns the exact backend payload. `Event` remains `Clone`,
`Debug`, `Eq`, and `Hash`, but is no longer `Copy`. Unknown future Crossterm
events degrade to `Event::Unknown`.

Single-line consumers should call `TextInputState::insert_str(&text)`. It
inserts at the current cursor, honors the existing grapheme limit, and consumes
only the prefix before the first CR or LF. Multi-line consumers own their own
policy; the future TextArea contract consumes the full normalized payload.

Insertion preserves the pasted payload's byte order at the original raw
insertion point, then repairs the cursor once against the globally segmented
post-edit value. Inserting a base before an existing combining mark, or a ZWJ
sequence between emoji, moves to the first valid boundary after the merged
grapheme instead of reordering later pasted scalars or leaving an invalid byte
offset inside it. Consumers that persisted cursor bytes must accept the
repaired boundary returned by `cursor_byte()`.

## Consumer edits

Before:

```rust
match event {
    Event::Paste => refresh(),
    other => route(other),
}
```

After:

```rust
match event {
    Event::Paste(text) => {
        let outcome = input.insert_str(&text);
        route_text_input(outcome);
    }
    other => route(other),
}
```

Code that reused an event after passing it by value must now borrow or clone it
explicitly:

```rust
inspect(&event);
route(event.clone());
audit(event);
```

Remove every `Event::Paste` unit pattern and replace it with
`Event::Paste(text)` or `Event::Paste(_)`. Do not create a compatibility event
or discard the payload in an adapter.

## Ownership changes

The neutral event owns pasted text because backend buffers do not outlive event
conversion. Widgets decide how much text they accept; consumers retain semantic
validation, submission, and effects.

## Removed concepts

- Unit paste markers with no recoverable payload.
- Implicit event duplication through `Copy`.
- Adapter-level paste-text discard.

No alias, compatibility facade, or parallel payload-free event remains.
