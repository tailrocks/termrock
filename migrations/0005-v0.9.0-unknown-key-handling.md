# 0005 — `v0.9.0` unknown key handling

## Boundary

`v0.9.0` represents backend keys outside TermRock's neutral vocabulary as
`KeyCode::Unknown`. Such keys and all key-release events are non-actionable.
No compatibility facade is provided.

## Old to new

| Before | New canonical surface | Required consumer edit |
|---|---|---|
| Unmodeled backend keys became `KeyCode::Esc` | Unmodeled backend keys become `KeyCode::Unknown` | Stop relying on function or media keys to trigger Escape behavior. |
| Exhaustive matches covered the original `KeyCode` variants | `KeyCode` also contains `Unknown` | Add `KeyCode::Unknown => /* ignore */`. |
| Some widgets processed key-release events | Text input, list, and choice dialog ignore releases consistently | Send press/repeat/release events unchanged; widgets own filtering. |

## Exhaustive key matches

Before:

```rust
match key.code {
    KeyCode::Esc => cancel(),
    // Existing variants.
}
```

After:

```rust
match key.code {
    KeyCode::Esc => cancel(),
    KeyCode::Unknown => {}
    // Existing variants.
}
```

## Removed concepts

- treating an unknown backend key as Escape;
- consumer workarounds that suppress function keys before TermRock;
- duplicate activation caused by processing key-release events.
