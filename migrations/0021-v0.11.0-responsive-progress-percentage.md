# 0021 ‚Äî `v0.11.0` responsive `Progress` percentage

## Boundary

Determinate `Progress` now reserves its textual percentage only when the
rendering area is at least 16 columns wide. Narrow areas keep the label and the
filled/empty glyph track, which remain non-color completion cues. An
indeterminate progress indicator configured with an empty frame slice is a
true no-op and does not alter buffer content or style.

## Old to new

| Previous behavior | Canonical behavior | Required consumer edit |
|---|---|---|
| Determinate progress always attempted to paint a percentage | Widths below 16 columns elide the percentage | Remove layouts that manually hide or replace narrow progress bars; render `Progress` directly at the available width. |
| Empty indeterminate frames still applied the muted style to the area | Empty frames leave the complete target buffer unchanged | Use `.frames(&[])` when a caller intentionally wants no progress paint. |

## After

```text

```

At 16 columns and wider, determinate progress continues to show its percentage.

## Ownership

`Progress` owns responsive allocation between label, glyph track, and
percentage. Consumers own the available layout rectangle, fraction, animation
tick, and optional custom frames.
