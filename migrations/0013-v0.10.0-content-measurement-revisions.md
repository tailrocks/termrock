# Migration 0013: content measurement revisions

Large borrowed widgets can now reuse content dimensions across cursor-only
repaints. Caching is opt-in because TermRock cannot detect same-length content
mutation through a borrowed slice.

## Canonical replacement

Pass a stable revision when constructing large `Viewport` or `DetailTable`
values, and bump it whenever content changes:

```rust
let viewport = Viewport::new(&lines, theme).content_revision(lines_revision);
let table = DetailTable::new(&rows, theme).content_revision(rows_revision);
```

Length changes invalidate automatically. Omitting `content_revision` preserves
the previous always-measure behavior. A stale revision can produce stale
scrollbar dimensions; domain content and revision ownership remain with the
consumer.

## State construction

`DialogScroll` and `DetailTableState` now carry private measurement caches.
Replace state literals with default construction plus mutation of the public
interaction fields:

```rust
// Before
let scroll = DialogScroll { scroll_x: 0, scroll_y: 12 };

// After
let mut scroll = DialogScroll::default();
scroll.scroll_y = 12;
```

Construct `DetailTableState` the same way:

```rust
let mut state = DetailTableState::default();
state.selected = Some(row_id);
```

Do not persist `Measured`, `row_offsets`, or the other measurement fields.
They are derived render caches. Persist domain selection and scroll offsets,
then rebuild the state with `Default`.

## New text helper

`text::display_cols_slice_into` writes a Unicode-safe display-column slice into
a reusable `String`. Existing callers of `display_cols_slice` need no changes.

## Removed concepts and ownership

No widget or helper was removed. Consumers still own content mutation and now
also own its revision counter when opting into caching. TermRock owns measured
dimensions, visible-row indices, scratch buffers, and invalidation on length or
layout changes.
