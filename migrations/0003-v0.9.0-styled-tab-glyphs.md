# 0003 — `v0.9.0` styled tab glyphs

## Boundary

`v0.9.0` makes tab glyph presentation caller-owned without requiring
consumers to reconstruct tab geometry or patch the rendered buffer.

## Old to new

| Before | New | Required consumer edit |
|---|---|---|
| `Tab { glyph: Option<&str>, .. }` | `Tab { glyph: Option<Span>, .. }` | Wrap plain glyphs with `Span::raw`; use `Span::styled` when the glyph conveys semantic state. |

## Consumer edit

Before:

```rust
Tab {
    glyph: Some("✓"),
    // ...
}
```

After:

```rust
Tab {
    glyph: Some(Span::styled("✓", theme.style(Role::Success))),
    // ...
}
```

The tab retains ownership of active, hover, focus, clipping, and pointer
geometry. The glyph span only overrides cells it paints, so the tab fill and
non-color selection cues remain intact.

## Removed concepts

- unstyled `&str` tab glyphs;
- consumer-side tab coordinate reconstruction for semantic glyph colors.
