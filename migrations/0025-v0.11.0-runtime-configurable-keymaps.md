## Removed surface

`KeyBinding` public-field literals and `Keymap::new` are removed. Binding fields
are private. `Keymap::glyph_for` no longer returns only `&'static str`; runtime
text requires `Cow<'_, str>`.

## Canonical replacement

There is one Cow-backed `Keymap`. Static defaults use
`KeyBinding::borrowed` and `Keymap::from_static`. Runtime-loaded bindings use
`KeyBinding::owned` and `Keymap::from_owned`. Dispatch, hints, glyph lookup,
axis filtering, mutation, and conflict diagnostics all read this same binding
slice.

Before:

```rust,ignore
static BINDINGS: &[KeyBinding<Action>] = &[KeyBinding {
    chords: &[KeyChord::plain(KeyCode::Char('q'))],
    action: Action::Quit,
    hint: Some("quit"),
    visibility: Visibility::Shown,
    glyph: None,
}];
let keymap = Keymap::new(BINDINGS);
```

After:

```rust,ignore
static BINDINGS: &[KeyBinding<Action>] = &[KeyBinding::borrowed(
    &[KeyChord::plain(KeyCode::Char('q'))],
    Action::Quit,
    Some("quit"),
    Visibility::Shown,
    None,
)];
let mut keymap = Keymap::from_static(BINDINGS);
keymap.remap(Action::Quit, vec![KeyChord::ctrl(KeyCode::Char('c'))]);
```

## Exact consumer edits

| Removed or changed surface | Consumer edit |
|---|---|
| `KeyBinding { chords, action, hint, visibility, glyph }` | Use `KeyBinding::borrowed(...)` for static data or `KeyBinding::owned(...)` for runtime data. |
| `Keymap::new(BINDINGS)` | Use `Keymap::from_static(BINDINGS)`. |
| Direct binding field reads | Use `chords()`, `action()`, `hint()`, `visibility()`, and `glyph()`. |
| `glyph_for(action) -> &'static str` | Accept `Cow<'_, str>`; compare through `==`, borrow with `.as_ref()`, or own with `.into_owned()`. |
| Application-owned dispatch/hint override tables | Delete them. Apply `remap`, `replace`, or `disable` to the one `Keymap`. |

`remap` replaces the chord set and clears any explicit glyph, preventing a
stale advertised shortcut. The canonical glyph then derives from the first new
chord. Use `replace` when a custom grouped glyph must remain explicit.

`conflicts()` reports each distinct-binding collision in declaration order.
It is diagnostic, not policy: `dispatch` remains first-binding-wins.

Under the `serde` feature, `KeyCode`, `KeyModifiers`, `KeyChord`, `Visibility`,
`KeyBinding`, and `Keymap` are serializable. Deserialization always creates
owned chord, text, and binding storage. TermRock does not own file discovery,
configuration merge policy, conflict UX, or action naming.

## Removed concepts

- Static-only keymap storage.
- Public mutation that could desynchronize binding invariants.
- Separate runtime override or hint registries.
- Explicit glyphs surviving a chord remap and advertising stale keys.

No compatibility constructor, public-field facade, or parallel runtime map
exists.
