# Migration 0012: widget construction and growth contracts

Every renderable widget now uses private fields, a `new(required...)`
constructor, and builder methods for optional configuration. Every widget also
implements both owned and borrowed Ratatui rendering traits.

## Construction replacements

Replace public-field literals with these canonical forms:

| Removed construction | Replacement |
|---|---|
| `ActionBar { actions, gap, theme }` | `ActionBar::new(actions, theme).gap(gap)` |
| `Backdrop { symbol, style }` | `Backdrop::new().symbol(symbol).style(style)` |
| `ChoiceDialog { dialog, actions, gap }` | `ChoiceDialog::new(dialog, actions).gap(gap)` |
| `DetailTable { rows, label_width, wrap, theme }` | `DetailTable::new(rows, theme).label_width(label_width).wrap(wrap)` |
| `Dialog { title, body, style, theme, emphasis }` | `Dialog::new(title, body, theme).style(style).emphasis(emphasis)` |
| `DiffView { lines, theme }` | `DiffView::new(lines, theme)` |
| `HintBar { hints, separator, theme }` | `HintBar::new(hints, theme).separator(separator)` |
| `List { rows, theme }` | `List::new(rows, theme)` |
| `LogPane { title, theme }` | `LogPane::new(theme).title(title)` |
| `MessageDialog { dialog, details, label_width, wrap, theme }` | `MessageDialog::new(dialog, details, theme).label_width(label_width).wrap(wrap)` |
| `Panel { title, emphasis, style, theme }` | `Panel::new(theme).title(title).emphasis(emphasis).style(style)` |
| `Progress { kind, label, theme }` | `Progress::new(kind, theme).label(label)` |
| `StatusBar { left, right, theme, alpha }` | `StatusBar::new(left, right, theme).alpha(alpha)` |
| `Tabs { tabs, gap, theme }` | `Tabs::new(tabs, theme).gap(gap)` |
| `TextInput { label, placeholder, validation, theme }` | `TextInput::new(label, theme).placeholder(placeholder).validation(validation)` |
| `Tree { nodes, theme }` | `Tree::new(nodes, theme)` |
| `Viewport { lines, title, theme, content_style }` | `Viewport::new(lines, theme).title(title).content_style(style)` |

`Form`, `SplitPane`, and `Toast` retain their existing constructors and
builders; their widget fields are now consistently private. `TextInput::label()`
exposes the configured semantic label without reopening field mutation.

`FormField` is now `#[non_exhaustive]`. Replace its struct literal with
`FormField::new(id, label, value)` and the `help`, `error`, `required`, and
`enabled` builders. Other projected row/config types remain exhaustive
pub-field structs because direct projection literals are their primary use.

## Non-exhaustive enums

External exhaustive matches must add a fallback arm for these growth-oriented
public enums:

- `style::Role`
- `input::{Event, KeyCode, MouseEventKind}`
- `interaction::Outcome`
- `osc::{PointerShape, Request}`
- `widgets::{Anchor, DetailTableOutcome, DiffKind, FormOutcome, PanelEmphasis,
  ProgressKind, RowRole, Severity, SplitPaneOutcome, TextInputOutcome,
  TreeNodeStatus, TreeOutcome}`

`KeyEventKind` and fixed two-state flags remain exhaustive.

## Owned rendering

All 20 widgets now accept both idioms:

```rust
frame.render_widget(Panel::new(theme), area);

let panel = Panel::new(theme);
frame.render_widget(&panel, area);
```

The same applies to owned and borrowed `StatefulWidget` values.

## Optional serialization

Enable `termrock`'s `serde` feature to serialize persistable plain-data state:

- `DiffState`
- `SplitRatio`
- `TextInputState`

`ListState`, `TreeState`, `FormState`, `SplitPaneState`, `LogPaneState`, and the
remaining interaction states intentionally do not implement serde: they contain
painted `Rect` regions, viewport geometry, styled lines, or other runtime
caches. Persist domain selection/input separately rather than serializing stale
render state.
