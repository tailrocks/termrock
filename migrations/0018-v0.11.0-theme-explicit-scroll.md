# 0018 — `v0.11.0` theme-explicit scroll and typed dialog input

## Boundary

`v0.11.0` removes the short-lived default-theme and raw-input compatibility
surfaces added after migration 0017. Scroll and dialog rendering now receives
the caller's semantic `Theme`; terminal protocols must cross the neutral input
adapter before shared state handles them. The canonical bordered line viewer is
`Viewport`, not a second free-function rendering body.

This boundary supersedes the APIs below. It does not alter migration 0017,
which remains the historical record of its earlier commit.

## Old to new

| Removed or changed surface | Canonical replacement | Required consumer edit |
|---|---|---|
| `DialogBorder::{Default, Danger}` | `PanelEmphasis::{Normal, Focused, Danger}` | Replace the duplicate dialog-only enum with the shared semantic chrome vocabulary. |
| `render_dialog_shell(frame, area, title, border)` | `render_dialog_shell(frame, area, title, emphasis, theme)` | Pass shared panel emphasis and the consumer's semantic theme. |
| `render_scrollable_dialog_body(..., scroll)` | `render_scrollable_dialog_body(..., scroll, theme)` | Pass the same theme used by the shell. |
| `DialogScroll::render_scrollbars(..., content_width)` | `DialogScroll::render_scrollbars(..., content_width, theme)` | Pass the surface theme. |
| `DialogScroll::on_mouse_scroll*` | `DialogScroll::handle_mouse` | Adapt terminal input to `MouseEventKind` + `KeyModifiers`, derive `ScrollAxes`, then call the canonical handler. |
| `DialogScroll::handle_raw_key_for_axes` | `DialogScroll::handle_key_for_axes` | Convert bytes through the terminal adapter to `KeyEvent`; shared widgets no longer parse escape bytes. |
| `DialogScroll::on_sgr_wheel_button_for_axes` | `DialogScroll::handle_mouse` | Decode SGR in the backend adapter, then pass the neutral mouse event. |
| `scroll::line_width` | `ratatui_core::text::Line::width` | Call the Ratatui API directly. |
| `dialog_inner_chunks`, `dialog_inner_height` | `layout::bottom_rows` plus the caller's body constraint | Describe the actual bottom-row heights instead of relying on a fixed five-slot facade. |
| `render_scrollable_block`, `render_scrollable_block_at` | `Viewport::new(lines, theme)` + `DialogScroll` | Render the canonical borrowed stateful widget; set `title` and `PanelEmphasis` through its builders. |
| `render_lines_with_offset_in_area(frame, area, lines, offset)` | same function with `&lines` and `theme` | Borrow line data and pass the surface theme. |
| public `render_*scrollbar*` helper family | `render_scrollbar(buffer, area, ScrollbarSpec::new(axis, geometry), theme)` | Submit one typed paint request for an explicit track; use `.style(ScrollbarStyle::Block)` when needed. |

## Before

```rust
use termrock::{
    layout::{DialogBorder, render_dialog_shell},
    scroll::{DialogScroll, ScrollAxes},
};

let inner = render_dialog_shell(frame, area, Some("Output"), DialogBorder::Default);
let mut scroll = DialogScroll::default();
let raw_key = b"\x1b[B";
let axes = ScrollAxes { vertical: true, horizontal: false };
scroll.handle_raw_key_for_axes(raw_key, axes);
```

## After

```rust
use termrock::{
    Theme,
    input::{KeyCode, KeyEvent, KeyModifiers},
    layout::render_dialog_shell,
    scroll::{DialogScroll, ScrollAxes},
    widgets::PanelEmphasis,
};

let theme = Theme::default();
let inner = render_dialog_shell(
    frame,
    area,
    Some("Output"),
    PanelEmphasis::Focused,
    &theme,
);
let mut scroll = DialogScroll::default();
// The terminal backend decodes bytes first; shared state receives neutral input.
let key_event = KeyEvent::new(KeyCode::Down, KeyModifiers::NONE);
let axes = ScrollAxes { vertical: true, horizontal: false };
scroll.handle_key_for_axes(key_event, 100, 20, 80, 80, axes);
```

Replace fixed five-slot geometry explicitly:

```rust
use ratatui_core::layout::Rect;
use termrock::layout::bottom_rows;

let inner = Rect::new(0, 0, 80, 20);
let content_rows = 8;
let old_minimum_height = 1 + content_rows + 1 + 1 + 1;

let (body, [status, validation, hints]) = bottom_rows(inner, [1, 1, 1]);
let spacer_height = body.height.min(1);
let spacer = Rect::new(body.x, body.y, body.width, spacer_height);
let content = Rect::new(
    body.x,
    body.y.saturating_add(spacer_height),
    body.width,
    body.height.saturating_sub(spacer_height),
);
let chunks = [spacer, content, status, validation, hints];
assert_eq!(old_minimum_height, 12);
```

For a bordered viewer, replace free-function painting with the canonical widget:

```rust
use ratatui_core::widgets::StatefulWidget;
use termrock::{
    Theme,
    scroll::DialogScroll,
    widgets::{PanelEmphasis, Viewport},
};

let theme = Theme::default();
let viewport = Viewport::new(&lines, &theme)
    .title("Output")
    .emphasis(PanelEmphasis::Focused);
let mut scroll = DialogScroll::default();
frame.render_stateful_widget(viewport, area, &mut scroll);
```

Replace wrapper-based scrollbar painting with the typed primitive:

```rust
use termrock::{
    Theme,
    scroll::{
        ScrollAxis, ScrollbarGeometry, ScrollbarSpec, ScrollbarStyle, render_scrollbar,
    },
};

let theme = Theme::default();
render_scrollbar(
    frame.buffer_mut(),
    track_area,
    ScrollbarSpec::new(
        ScrollAxis::Vertical,
        ScrollbarGeometry::new(content_height, viewport_height, scroll_y),
    )
    .style(ScrollbarStyle::Block),
    &theme,
);
```

## Ownership changes

TermRock owns semantic scroll geometry, neutral input handling, and canonical
viewport rendering. Consumers own the active theme, terminal-protocol adapter,
content, and choice of normal, focused, or danger emphasis.

## Removed concepts

- implicit `Theme::default()` inside reusable render helpers;
- raw escape-byte and SGR-button parsing in shared widget state;
- duplicate bordered scroll rendering beside `Viewport`;
- fixed five-slot dialog compatibility geometry;
- aliases that duplicate `handle_mouse`.
- parallel dialog-border emphasis outside `Panel`;
- a wrapper family around one scrollbar render primitive.

## Why this is stronger

Every brand now reaches dialog bodies and scrollbars through semantic roles.
One typed input path prevents terminal-protocol parsing from diverging across
widgets. One canonical `Viewport` body removes duplicated clipping, focus,
Unicode, measurement, and scrollbar behavior.
