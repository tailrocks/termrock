# 0011 — `v0.10.0` trailing metadata cells

## Boundary

`v0.10.0` makes aligned trailing metadata part of canonical List and Tree row
rendering. Consumers no longer calculate local columns or patch rendered
buffers.

## Old to new

| Before | New canonical surface | Required consumer edit |
|---|---|---|
| `ListRow { id, label, role, enabled }` | `ListRow { id, label, trailing, role, enabled }` | Add `trailing: None`, or supply `Some(Line)` for right-aligned metadata. |
| `TreeNode { id, label, depth, .. }` | `TreeNode { id, label, trailing, depth, .. }` | Add `trailing: None`, or supply `Some(Line)` for right-aligned metadata. |
| Exhaustive `TreeOutcome` matches | New `CheckToggled(Id)` variant | Handle checked-state changes separately from row activation and tree disclosure. List reports `Outcome::Changed`; read its `Selection` for checked identities. |
| `ListState` struct literals | `ListState::new(selected)` or `ListState::default()` | Replace struct literals; multi-select bookkeeping is widget-owned. |
| Consumer-owned label/metadata width math | Widget-owned reserved trailing column | Delete local padding, truncation, and buffer patching. |

## Consumer edit

Before:

```rust
ListRow {
    id: "entry",
    label: Line::from("Example"),
    role: RowRole::Item,
    enabled: true,
}
```

After:

```rust
ListRow {
    id: "entry",
    label: Line::from("Example"),
    trailing: Some(Line::from("12 KiB")),
    role: RowRole::Item,
    enabled: true,
}
```

The widest trailing value defines a reserved column. Values align to its right
edge; labels truncate first on narrow terminals. When the trailing cell itself
cannot fit, it is Unicode-safely clipped to the available width.

Multi-select is opt-in through `enable_multi_select()`. Read or mutate the
ordered checked identities through `selection()` / `selection_mut()`. Existing
single-selection behavior remains unchanged while multi-select is disabled.

Exhaustive Tree outcome matches must add `CheckToggled(id)`. List follows the
canonical shared event contract and reports `Outcome::Changed` after a check:

```rust
match tree_outcome {
    TreeOutcome::CheckToggled(id) => persist_checked(id),
    // existing outcome arms
    _ => {}
}
```

## Removed concepts

- consumer-computed metadata padding;
- local right-column width calculations;
- post-render buffer patches for sizes, counts, or durations.
