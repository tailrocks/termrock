# Migration 0023: List multi-select contract alignment

## Removed surface

Every field of `ListState<Id>` was publicly mutable. Consumers could mutate
semantic state through two competing paths and could corrupt render-owned
`regions` and `check_regions` geometry. List check gestures also returned bare
`Outcome::Changed`, unlike Tree's identity-bearing result.

## Canonical replacement

`ListState` now owns all fields privately. Use its semantic methods:

- `selected()` and `select(...)` for the keyboard cursor;
- `hovered()` and `hover(...)` for pointer state;
- `is_focused()` and `set_focused(...)` for focus ownership;
- `offset()`, `scroll_by(...)`, and `scroll_to_position(...)` for scrolling;
- `regions()` for read-only painted item geometry;
- `selection()` and `selection_mut()` for ordered checked identities.

Checkbox hit geometry remains frame-internal and has no accessor. Shared
`Outcome<T>` gains `CheckToggled(T)`. Both keyboard Space and a painted
checkbox click now return the toggled stable identity.

This supersedes migration 0011's statement that List reports
`Outcome::Changed` for a check gesture. Tree remains unchanged and continues to
return `TreeOutcome::CheckToggled(id)`.

## Consumer edits

Replace field access and struct literals with semantic construction and
methods.

Before:

```rust
let mut state = ListState {
    selected: Some("job-7"),
    focused: false,
    ..ListState::default()
};
state.selection.as_mut().unwrap().toggle(&"job-7");
let selected = state.selected.as_ref();
```

After:

```rust
let mut state = ListState::new(Some("job-7"));
state.set_focused(false);
state.enable_multi_select();
state.selection_mut().unwrap().toggle(&"job-7");
let selected = state.selected();
```

Handle check outcomes without rescanning the complete checked collection:

```rust
match state.handle_key(&rows, key) {
    Outcome::CheckToggled(id) => persist_check(id, state.selection()),
    Outcome::Activated(id) => open(id),
    _ => {}
}
```

Delete writes to `offset`, `viewport_height`, `regions`, or `check_regions`.
Render and interaction methods own those invariants. Read painted item geometry
through `regions()` only. No check-region replacement exists because consumers
must not own frame-internal checkbox geometry.

## Ownership changes

`ListState` exclusively owns cursor, hover, focus, scroll, multi-select, and
painted hit-geometry invariants. Consumers choose domain effects after reading
identity-bearing outcomes. This matches Tree's state-visibility and gesture
reporting model while retaining the shared `Outcome<T>` vocabulary.

## Removed concepts

- Publicly mutable List state fields.
- Consumer mutation of frame-internal hit geometry.
- Bare `Outcome::Changed` for List checkbox gestures.
- Rescanning `Selection::checked()` to discover which identity toggled.

No deprecated fields, compatibility facade, duplicate outcome type, or
payload-free check result remains.
