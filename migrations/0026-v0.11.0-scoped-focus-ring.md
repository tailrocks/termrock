# 0026: Scoped per-frame focus ring

TermRock now owns inter-widget traversal, dynamic-target reconciliation,
pointer focus, modal trapping, and opener restoration through
`interaction::FocusRing<Id, ScopeId>`.

The narrower `FocusState`, screen-specific `FocusOwner`, and `ButtonFocus`
surfaces are removed. They could not represent dynamic registration or keep
modal lifecycle and focus scope changes atomic.

## Consumer edits

Before, callers retained a focus cell or a fixed ring and separately changed a
`ModalStack`:

```rust
let mut focus = FocusState::new(Some(ScreenId::Sidebar));
focus.set(Some(ScreenId::Preview));
modals.open(DialogId::Confirm);
```

After, register current layout truth each frame and coordinate modal lifecycle
through the ring:

```rust
let mut focus = FocusRing::new(ScopeId::Screen, Some(ScreenId::Sidebar));

focus.begin_frame();
focus.register_order(
    ScopeId::Screen,
    [
        (ScreenId::Sidebar, Some(sidebar_area), true),
        (ScreenId::Preview, Some(preview_area), true),
    ],
);
focus.reconcile();

focus.request_focus(ScreenId::Preview);
focus.open_modal(&mut modals, DialogId::Confirm, ScopeId::Confirm);
```

Use `handle_key` for Tab/BackTab, `focus_at` for pointer transfer,
`is_focused` to gate widget input, and `panel_emphasis_for` to project the
single focused container into semantic border emphasis. Composite widgets may
register `area: None` before rendering and call `attach_region` with their
canonical painted geometry afterward.

Close modal flows only through `pop_modal` or `clear_modals`; calling
`ModalStack` lifecycle mutators directly is no longer available. Its
`from_current`, `from_parts`, `into_parts`, `parents_mut`, `open`, `open_sub`,
`pop`, `clear_chain`, and `take_current` public operations are removed so the
modal chain and focus scope cannot diverge. `current`, `current_mut`,
`parents`, and the depth predicates remain available for projection and input.

## Removed concepts

- `FocusState<Id>`
- `FocusOwner<Tab>`
- `ButtonFocus`
- lookbook-local focus registry and independently paired modal scope calls

Form keeps its internal field focus, and List/Tree keep row selection. Register
each composite widget once in the screen ring; the ring coordinates between
widgets rather than retaining a widget tree.

### Replacing `FocusOwner`

```rust
// before
let emphasis = owner.panel_emphasis_for(&ScreenTab::Logs);
let show_cursor = owner.show_cursor_for(&ScreenTab::Logs);

// after
focus.register(FocusTarget {
    id: ScreenId::Logs,
    scope: ScopeId::Screen,
    area: Some(logs_area),
    enabled: true,
});
let emphasis = focus.panel_emphasis_for(&ScreenId::Logs);
let show_cursor = focus.is_focused(&ScreenId::Logs);
```

### Replacing `ButtonFocus`

```rust
// before
button_focus = button_focus.next();
button_focus = button_focus.prev();

// after
focus.register_order(
    ScopeId::Confirm,
    actions.iter().map(|action| (action.id, None, action.enabled)),
);
focus.handle_key(key_event); // plain Tab or BackTab
```

After rendering an `ActionBar` or `ChoiceDialog`, attach each state-produced
action region with `attach_region`; do not reconstruct label/gap geometry.
