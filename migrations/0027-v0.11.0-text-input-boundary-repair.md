# 0027 — TextInput insertion boundary repair

## Removed behavior

`TextInputState` could leave `cursor_byte()` inside an extended grapheme when
an inserted scalar merged with adjacent combining or ZWJ content.

## Canonical replacement

Insertion now repairs the cursor against the globally segmented post-edit
line. Consumers must treat `cursor_byte()` as a grapheme boundary, including
after inserting before a leading combining mark.

## Consumer edits

Code that asserted the old intermediate byte must update its expected cursor
to the first grapheme boundary at or after the logical insertion end. No API
name changes are required.

Before:

```rust
// Inserting `e` before U+0301 could report byte 1.
```

After:

```rust
// `e\u{301}` is one grapheme; the cursor reports byte 3.
```

## Removed concepts and ownership

Scalar-length cursor advancement is removed. The private shared edit core owns
grapheme boundary repair for both `TextInput` and `TextArea`.

## Validation

```sh
mise run gate
cargo test -p termrock text_input --all-features
cargo test -p termrock text_area --all-features
```
