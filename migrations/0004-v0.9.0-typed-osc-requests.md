# 0004 — `v0.9.0` typed OSC requests

## Boundary

`v0.9.0` closes caller-controlled OSC 52 selection vocabulary and rejects
unsafe OSC 8 hyperlinks or oversized clipboard writes. No compatibility facade
is provided.

## Old to new

| Before | New canonical surface | Required consumer edit |
|---|---|---|
| `ClipboardWrite { selection: "c", text }` | `ClipboardWrite { selection: ClipboardSelection::Clipboard, text }` | Replace selection strings with a typed variant. |
| Any hyperlink URL passed through verbatim | Only `http`, `https`, `mailto`, and `file` schemes encode; terminal controls are percent-encoded | Treat an empty encoded vector as rejection and emit nothing. |
| Clipboard writes had no payload bound | Source text above 100,000 bytes is rejected | Treat an empty encoded vector as rejection and emit nothing. |

## Clipboard selection

Before:

```rust
let request = ClipboardWrite { selection: "c", text };
```

After:

```rust
let request = ClipboardWrite {
    selection: ClipboardSelection::Clipboard,
    text,
};
```

## Rejected requests

```rust
let bytes = encode_hyperlink_open(None, url);
if !bytes.is_empty() {
    writer.write_all(&bytes)?;
}
```

## Removed concepts

- free-form OSC 52 selection strings;
- raw caller-controlled strings in OSC 8 terminal output;
- unbounded OSC 52 clipboard writes;
- emitting rejected requests.
