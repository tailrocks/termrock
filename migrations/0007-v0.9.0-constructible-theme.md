# 0007 — `v0.9.0` constructible theme

## Boundary

`v0.9.0` makes every styled widget consume semantic `Theme` roles. Widget
chrome no longer accepts disconnected raw styles, and hint rendering now
requires a theme. No compatibility fields or overloads are provided.

## Old to new

| Removed surface | Canonical replacement | Required consumer edit |
|---|---|---|
| `Tabs` without `theme` | `Tabs { theme: &theme, .. }` | Supply a borrowed `Theme`. |
| `ActionBar` without `theme` | `ActionBar { theme: &theme, .. }` | Supply a borrowed `Theme`; per-action overrides remain. |
| `HintBar` without `theme` | `HintBar { theme: &theme, .. }` | Supply a borrowed `Theme`. |
| `StatusBar.style` | `StatusBar.theme` and `Role::StatusBar` | Move the base style into a theme override. |
| `DiffView.added_style` / `removed_style` | `DiffView.theme` and `Role::DiffAdded` / `DiffRemoved` | Override semantic diff roles. |
| `Viewport` chrome style fields | `Viewport.theme` | Supply a theme; use `content_style: Some(_)` only for one-off content styling. |
| `render_hint_bar(frame, area, spans)` | `render_hint_bar(frame, area, spans, theme)` | Pass the active theme. |
| `styled_hint_spans(spans, remap)` | `styled_hint_spans(spans, theme, remap)` | Pass the active theme. |
| `wrapped_hint_lines(spans, width)` | `wrapped_hint_lines(spans, width, theme)` | Pass the active theme. |

## Theme construction

Build a complete theme with `Theme::from_fn`, or start from a preset and
override selected roles with `Theme::with_role`. `Theme::roles` exposes every
role in stable positional order for inspection and complete transformations.

Before:

```rust
let view = DiffView {
    lines: &lines,
    added_style: Style::new().bold(),
    removed_style: Style::new().dim(),
};
```

After:

```rust
let theme = Theme::default()
    .with_role(Role::DiffAdded, Style::new().bold())
    .with_role(Role::DiffRemoved, Style::new().dim());
let view = DiffView {
    lines: &lines,
    theme: &theme,
};
```

## Ownership changes

TermRock owns semantic role selection for tabs, hints, actions, status bars,
diffs, and viewport chrome. Consumers own palette construction and deliberate
per-instance escape hatches. Rendering behavior and interaction state remain
widget-owned.

## Removed concepts

- widget-specific raw chrome palettes;
- hardcoded phosphor colors inside widget renderers;
- hint styling detached from the active theme.
