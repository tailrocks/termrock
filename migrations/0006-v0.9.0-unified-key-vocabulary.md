# 0006 — `v0.9.0` unified key vocabulary

## Boundary

`v0.9.0` removes duplicate keymap key and modifier types. `KeyChord` now uses
the backend-neutral input vocabulary directly. No compatibility aliases are
provided.

## Old to new

| Before | New canonical surface | Required consumer edit |
|---|---|---|
| `keymap::LogicalKey` | `input::KeyCode` | Rename imports and variants. |
| `keymap::Mods` | `input::KeyModifiers` | Rename imports and use `CONTROL` instead of `CTRL`. |
| `KeyChord { key: LogicalKey, mods: Mods }` | `KeyChord { key: KeyCode, mods: KeyModifiers }` | Use input types in direct struct construction. |

## Key chords

Before:

```rust
use termrock::keymap::{KeyChord, LogicalKey, Mods};
const QUIT: KeyChord = KeyChord::ctrl(LogicalKey::Char('q'));
```

After:

```rust
use termrock::input::KeyCode;
use termrock::keymap::KeyChord;
const QUIT: KeyChord = KeyChord::ctrl(KeyCode::Char('q'));
```

## Modifier bit layout

Code that persisted raw modifier bits must migrate values rather than
reinterpreting them. `CONTROL` moved from bit 0 to bit 1; `SHIFT` moved from
bit 2 to bit 0; `ALT` moved from bit 1 to bit 2. Named constants and
`with_ctrl`, `with_alt`, and `with_shift` remain the canonical construction
path.

## Removed concepts

- `keymap::LogicalKey`;
- `keymap::Mods`;
- mirror matches converting one neutral key vocabulary into another;
- `LogicalKey::Char('\0')` as an unknown-key sentinel.
