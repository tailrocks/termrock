# 0009 — `v0.9.0` neutral event contract

## Boundary

`v0.9.0` establishes one backend-neutral interaction contract: handlers live
on widget state, borrowed widget data precedes the event, pointer actions use a
shared naming family, and generic outcomes use `interaction::Outcome`. No
compatibility methods, aliases, or backend event re-exports remain.

## Event vocabulary

`input::Event`, `input::MouseEvent`, and `input::MouseButton` now represent
keys, mouse buttons and motion, focus, resize, paste markers, and unknown
backend events. Crossterm events convert with `From`:

```rust
let event = termrock::input::Event::from(crossterm_event);
```

The removed `termrock::crossterm::{Event, KeyEvent, MouseEvent, key,
mouse_kind}` surface has no replacement alias. Import neutral types from
`termrock::input`; mention Crossterm types only at the backend boundary.

## Handler changes

| Before | Canonical replacement |
|---|---|
| `ChoiceDialogState::handle_key(key, actions)` | `state.handle_key(actions, key)` |
| `ChoiceDialogState::activate_at(position)` | `state.click(position)` |
| `StatusBarState::activate_at(position)` | `state.click(position)` |
| `DetailTableState::hover_at(position)` | `state.hover(position)` |
| `DetailTableState::activate_at(position)` | `state.click(position)` |
| `DetailTableState::activate_link_at(position)` | `state.click_link(position)` |
| `DetailTable::outcome_at(state, position)` | `state.click(position)` |
| `DetailTable::activate_selected(state)` | `state.activate_selected(rows)` |
| no DetailTable keyboard handler | `state.handle_key(rows, key)` |
| `SplitPane::handle_key(state, key)` | `state.handle_key(&split, key)` |
| `SplitPane::pointer_down(state, position)` | `state.drag_start(&split, position)` |
| `SplitPane::pointer_move(state, position)` | `state.hover(&split, position)` |
| `SplitPane::pointer_drag(state, position)` | `state.drag_move(&split, position)` |
| `SplitPane::pointer_up(state)` | `state.drag_end()` |
| `SplitPane::collapse(state, side)` | `state.collapse(side)` |
| `SplitPane::expand(state)` | `state.expand()` |
| `TreeState::scroll_by(i32, len) -> usize` | `state.scroll_by(isize, len) -> bool` |
| `FormState::scroll_by(i32) -> usize` | `state.scroll_by(isize, content_len) -> bool` |

Keyboard wiring before:

```rust
let outcome = dialog.handle_key(key, &actions);
```

After:

```rust
let outcome = dialog.handle_key(&actions, key);
```

Mouse wiring before:

```rust
let outcome = split.pointer_down(&mut state, position);
```

After:

```rust
let outcome = state.drag_start(&split, position);
```

## Outcome changes

`ListOutcome<Id>` is removed; List returns `interaction::Outcome<Id>`, also
re-exported as `widgets::Outcome`. `ModalOutcome<T>` is removed without a
replacement because it had no component contract or consumers. Widget-specific
outcomes remain where their variants carry distinct semantics.

## DetailTable contract

DetailTable now owns Up/Down and j/k selection plus Enter activation through
`DetailTableState::handle_key`. Its catalog keyboard contract changes from
caller-owned to covered. Rendering and pointer behavior are unchanged.
