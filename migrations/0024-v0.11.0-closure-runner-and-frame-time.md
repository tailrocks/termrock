# 0024 — Closure runner and immutable frame time

## Removed surface

The speculative `runtime::Component`, `View`, `Dirty`, `UpdateResult`,
`NoEffect`, `drive_frame`, `drive_render`, and subscription types are removed.
They were not used by TermRock's real application loop and incorrectly claimed
ownership of consumer effects and scheduling.

## Canonical replacement

Enable `crossterm` and call `runtime::run` with mutable render and neutral-event
update closures. Both callbacks receive the same immutable `FrameTick` sampled
once per frame. A final deadline callback lets timed state wake the loop before
the idle timeout. `ToastState` provides deterministic lifetime state.

Before:

```rust,ignore
let result = component.handle_event(&event);
if result.is_dirty() {
    drive_frame(&mut terminal, &view, &model, area, |_| {})?;
}
```

After:

```rust,ignore
use std::ops::ControlFlow;
use termrock::runtime::{RunOptions, run};

run(
    &mut model,
    RunOptions::default(),
    |model, frame, tick| model.render(frame, tick),
    |model, event, tick| {
        model.update(event, tick);
        ControlFlow::Continue(())
    },
    |model| model.next_deadline(),
)?;
```

## Exact surface mapping

| Removed item | Consumer edit |
|---|---|
| `Component::handle_event` | Move routing into the `runtime::run` update closure. Return `ControlFlow::Break(())` to quit. |
| `View::render` | Move projection into the mutable render closure; the runner supplies `&mut Frame` and `FrameTick`. |
| `drive_frame` / `drive_render` | Delete the manual `Terminal::draw` driver and call `runtime::run`. If a consumer intentionally owns another backend or loop, call that backend's `Terminal::draw` directly. |
| `Dirty` | Delete it. The synchronous runner redraws on each idle/event cycle. Timed work reports its next deadline. |
| `UpdateResult<Effect>` / `NoEffect` | Return `ControlFlow<()>` only for loop lifetime. Keep domain effects in the application and dispatch them from its update closure. |
| `SubscriptionPoll`, `Subscription`, `ClosureSubscription`, `StdSubscription` | Delete adapters. Poll external receivers/sources in consumer-owned infrastructure and route resulting neutral events or domain messages through application update policy. TermRock provides no replacement scheduler. |

The runner callback arity changed from the local four-argument prototype to
five arguments: add `|model| model.next_deadline()`, or `|_| None` when the
application has no timed state.

For a state-managed toast, replace an application-owned `shown_at`/TTL pair:

```rust
use std::time::Duration;
use termrock::widgets::{ToastLifetime, ToastState};

let mut state = ToastState::new(
    ToastLifetime::ExpiresAfter(Duration::from_secs(2)),
);
state.show(tick);
if state.is_visible(tick) { /* render Toast */ }
let deadline = state.next_deadline();
```

Consumers continue to own domain messages, effects, executors, external event
sources, and quit policy. Widgets never read the system clock; inject
`FrameTick::manual` in deterministic tests.
